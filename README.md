## Анализ

После открытия исполняемого файла в IDA была найдена функция проверки серийного номера:

```c
int __fastcall sub_4017BD(HWND a1, символ *a2)
```

Функция:

1. Проверяет формат ввода:

```
ХХХХХХХ-ХХХХХХ-ХХХХХХ-ХХХХХХ
```

2. Разбирает четыре шестнадцатеричных числа:

```c
v5 = стртул(...)
v6 = стртул(...)
v7 = стртул(...)
v8 = стртул(...)
```

3. Выполняет цикл `v8` раз:

```c
v12 = (v6^v5)+986598842;
v13 = (v12 ^ (v7 + 946341097)) + v10;
v14 = v13 ^ (v12 >> 3);

v5 = (v14 + 1670772768) ^ 0xE93E7D9A;
v6 = (v13 + (v10 ^ v6)) ^ 0x4E192BBA;
v7 = (v12 ^ (v14 + 1670772768)) + 1594596065;
```

4. После завершения цикла выполняется сравнение:

```c
если (
    dword_404024 == v5 &&
    dword_404020 == v6 &&
    dword_40401C == v7 &&
    dword_404018 == v8
)
```

## Извлечение целевых констант

В секции `.данные` по адресам `404018–404024` были обнаружены значения:

| Адрес  | Значение |
| ------ | -------- |
| 404018 | 00000F02 |
| 40401С | F34A28A0 |
| 404020 | F9C92FE1 |
| 404024 | 27C51458 |

Следовательно, проверка ожидает:

```text
v5 = 27C51458
v6 = F9C92FE1
v7 = F34A28A0
v8 = 00000F02
```

## Восстановление ключа

Был выполнен обратный проход алгоритма на:

```

```

итерации.

В результате получен серийный номер:

```text
F5821D13-118B9F18-D1EE9E68-00000F02
```
