Программа пытается открыть следующий подключ:


Второй вызов читает само значение.

# 4. Анализ алгоритма проверки

## Функция sub_4017D2

Функция выполняет Base64-кодирование строки.

Внутри присутствует собственный Base64-алфавит, который восстанавливается XOR-операцией над константным массивом.

После восстановления получается стандартный алфавит Base64:

```text
ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/
```

Следовательно:

```text
sub_4017D2(input) = Base64(input)
```

---

## Функция sub_401679

Функция выполняет 12 раундов XOR-преобразований.

Для каждого раунда строится ключ длиной 10 байт:

```python
key[i] =
    unk_405060[start+i]
    XOR
    unk_4050E0[start+i]
```

После чего каждый байт строки циклически XOR-ится с ключом.

Используется следующий порядок раундов:

```python
ROUNDS = [1, 4, 8, 3, 9, 11, 5, 10, 7, 2, 0, 6]
```

---

## Функция sub_401706

Функция рекурсивно вызывает XOR-преобразование и формирует эталонную последовательность байтов, которая впоследствии сравнивается с вычисленным результатом.

Проверка фактически выглядит следующим образом:

```text
Registry Value
        ↓
      Base64
        ↓
12 XOR-раундов
        ↓
Сравнение с byte_404020
```

При совпадении выводится сообщение:

```text
FLAG
```

## 6. Зачем дважды вызывается RegQueryValueExA?

Причина — получение данных переменного размера.

Первый вызов:

```c
RegQueryValueExA(..., NULL, &cbData);
```

используется для определения необходимого размера буфера.

Второй вызов:

```c
RegQueryValueExA(..., lpData, &cbData);
```

читает данные в выделенный буфер.

Это стандартный шаблон работы с WinAPI при чтении значений реестра неизвестной длины.

## Ход работы

После открытия файла в IDA Pro была исследована функция проверки серийного номера `sub_4017BD`.

В ходе анализа установлено, что программа ожидает ввод ключа следующего формата:

```text

```

Каждый блок представляет собой шестнадцатеричное число. После проверки формата введённые данные преобразуются в четыре 32-битных значения:

```c
v5 = strtoul(...);
v6 = strtoul(...);
v7 = strtoul(...);
v8 = strtoul(...);
```

Далее программа выполняет серию арифметических и логических преобразований в цикле длиной `v8` итераций:

```c
v12 = (v6 ^ v5) + 986598842;
v13 = (v12 ^ (v7 + 946341097)) + v10;
v14 = v13 ^ (v12 >> 3);

v5 = (v14 + 1670772768) ^ 0xE93E7D9A;
v6 = (v13 + (v10 ^ v6)) ^ 0x4E192BBA;
v7 = (v12 ^ (v14 + 1670772768)) + 1594596065;
```

После завершения цикла производится сравнение полученных значений с константами, расположенными в секции данных программы:

```c

    return value & 0xFFFFFFFF

    v6 = 0xF9C92FE1
    v7 = 0xF34A28A0
    v8 = 0x00000F02

        v12 = ctypes_int32(v7 + 730913689) ^ tmp_x
        v14 = ctypes_int32(tmp_x + 1084439911)
        v13 = ctypes_int32(v14 ^ (v12 >> 3))

            ctypes_int32(v6 ^ 0x3B8282EE) - v13
        )

        v5_old = ctypes_int32(
            ctypes_int32(v12 - 513126785) ^ v6_old
        )

            ctypes_int32(v13 ^ v12) - 989527920
        )

        v5 = v5_old
        v6 = v6_old
        v7 = v7_old

if __name__ == "__main__":
```
